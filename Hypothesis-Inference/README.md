# Hypothesis and Inference in Data Analysis

## Overview

It allows us to draw conclusions about a population based on a sample of data. This repository explores the fundamental concepts of hypothesis testing and statistical inference, with practical Python implementations for machine learning model selection.

## Introduction to Statistical Inference

**Statistical inference** is defined as the process of analyzing data generated by a sample, but then used to determine some characteristic of the larger population. The two main branches of statistics are:

1. **Descriptive Statistics**: Describe and summarize the data
2. **Inferential Statistics**: Make predictions and generalizations from the data

### Key Approaches to Statistical Inference

1. **Frequentist Approach**: Specifies probability distributions based on the frequency of certain data occurrences
2. **Bayesian Approach**: Incorporates prior knowledge or beliefs and updates them based on observed evidence
3. **Likelihood Approach**: Calculates parameter values most likely to produce observed distributions

### Components of Statistical Inference

The following components are essential for making statistical inferences:

- **Sample size** ($n$)
- **Variability in the sample** (standard deviation $\sigma$ or $s$)
- **Size of the observed differences**

## Hypothesis Testing Fundamentals

**Hypothesis testing** provides a formal framework for using sample data to decide whether a statement about a population is likely true or not. It's a method of making decisions about the parameters of a population based on random sampling.

### Mathematical Framework

The fundamental question in hypothesis testing: Does the observed data provide sufficient evidence to reject a claim about the population?

**Test Statistic Formula:**

For a population mean with known variance:
$$Z = \frac{\bar{x} - \mu_0}{\sigma / \sqrt{n}}$$

Where:
- $\bar{x}$ = sample mean
- $\mu_0$ = hypothesized population mean
- $\sigma$ = population standard deviation
- $n$ = sample size

### The Five-Step Process

**Step 1:** Set up the null hypothesis ($H_0$) by correctly identifying whether it is left-tailed, right-tailed, or two-tailed hypothesis testing.

**Step 2:** Set up the alternative hypothesis ($H_A$ or $H_1$).

**Step 3:** Choose the correct significance level, $\alpha$, and find the critical value.
- Common values: $\alpha = 0.05$ (5%) or $\alpha = 0.01$ (1%)

**Step 4:** Calculate the appropriate test statistic ($z$, $t$, or $\chi^2$) and p-value.

**Step 5:** Compare the test statistic with the critical value or compare the p-value with $\alpha$ to arrive at a conclusion.

**Decision Rule:**
- If $p\text{-value} < \alpha$: Reject $H_0$
- If $p\text{-value} \geq \alpha$: Fail to reject $H_0$

## Types of Hypotheses

### Null Hypothesis ($H_0$)

The **null hypothesis** is a mathematical statement used to indicate that there is no difference between two possibilities. It represents the status quo or no effect.

**Example:** $H_0: \mu_1 = \mu_2$ (The population means are equal)

### Alternative Hypothesis ($H_A$ or $H_1$)

The **alternative hypothesis** indicates that there is a statistical significance between two possible outcomes. It represents what we're trying to prove.

**Forms:**
- **Two-tailed:** $H_A: \mu \neq \mu_0$
- **Right-tailed:** $H_A: \mu > \mu_0$
- **Left-tailed:** $H_A: \mu < \mu_0$

### Types of Errors

| Reality | Decision: Reject $H_0$ | Decision: Fail to Reject $H_0$ |
|---------|------------------------|--------------------------------|
| $H_0$ is True | **Type I Error** ($\alpha$) | Correct Decision |
| $H_0$ is False | Correct Decision | **Type II Error** ($\beta$) |

- **Type I Error (False Positive, $\alpha$)**: Rejecting a true null hypothesis
  - Probability: $P(\text{Reject } H_0 | H_0 \text{ true}) = \alpha$
  
- **Type II Error (False Negative, $\beta$)**: Failing to reject a false null hypothesis
  - Probability: $P(\text{Fail to reject } H_0 | H_0 \text{ false}) = \beta$
  
- **Statistical Power**: $1 - \beta$ (Probability of correctly rejecting a false $H_0$)

## Statistical Tests

### 1. Z-Test (Large Sample Test)

**Used when:** $n \geq 30$ and population standard deviation ($\sigma$) is known

**Test Statistic:**
$$Z = \frac{\bar{x} - \mu_0}{\sigma / \sqrt{n}}$$

The Z-statistic follows a standard normal distribution $N(0, 1)$.

### 2. T-Test (Small Sample Test)

**Used when:** $n < 30$ and population standard deviation is unknown

**Test Statistic:**
$$t = \frac{\bar{x} - \mu_0}{s / \sqrt{n}}$$

Where $s$ is the sample standard deviation. The t-statistic follows a t-distribution with $df = n - 1$ degrees of freedom.

#### Variants of T-Test:

**One-Sample T-Test:** Compare sample mean to a known value

**Independent Two-Sample T-Test:**
$$t = \frac{\bar{x}_1 - \bar{x}_2}{\sqrt{\frac{s_1^2}{n_1} + \frac{s_2^2}{n_2}}}$$

**Paired T-Test (Dependent Samples):**
$$t = \frac{\bar{d}}{s_d / \sqrt{n}}$$

Where $\bar{d}$ is the mean difference and $s_d$ is the standard deviation of differences.

### 3. Chi-Square Test ($\chi^2$)

**Used when:** Testing independence between categorical variables

**Test Statistic:**
$$\chi^2 = \sum \frac{(O_i - E_i)^2}{E_i}$$

Where:
- $O_i$ = Observed frequency
- $E_i$ = Expected frequency

### 4. ANOVA (Analysis of Variance)

**Used when:** Comparing means of three or more groups

**F-Statistic:**
$$F = \frac{\text{Between-group variability}}{\text{Within-group variability}} = \frac{MS_{\text{between}}}{MS_{\text{within}}}$$

### 5. McNemar's Test

**Used when:** Comparing two classifiers on the same test set (paired nominal data)

**Test Statistic (with continuity correction):**
$$\chi^2 = \frac{(|b - c| - 1)^2}{b + c}$$

Where $b$ and $c$ are the counts of discordant pairs in the 2×2 contingency table:

|  | Model B Correct | Model B Wrong |
|---|-----------------|---------------|
| **Model A Correct** | a | b |
| **Model A Wrong** | c | d |

- **Null Hypothesis ($H_0$)**: Classifiers have similar proportion of errors
- **Alternative Hypothesis ($H_1$)**: Classifiers have different proportion of errors

### 6. Shapiro-Wilk Test

**Used when:** Testing if data follows a normal distribution (prerequisite for parametric tests)

- $H_0$: The sample has a Gaussian distribution
- $H_1$: The sample does not have a Gaussian distribution

## Confidence Intervals

A **confidence interval** provides a range of values that likely contains the true population parameter.

### Formula for Mean (Known $\sigma$):
$$CI = \bar{x} \pm Z_{\alpha/2} \cdot \frac{\sigma}{\sqrt{n}}$$

### Formula for Mean (Unknown $\sigma$):
$$CI = \bar{x} \pm t_{\alpha/2, df} \cdot \frac{s}{\sqrt{n}}$$

**Interpretation:** A 95% confidence interval means that if we repeated the sampling process many times, about 95% of the constructed intervals would contain the true population parameter.

### Relationship to Hypothesis Testing

The confidence level and significance level are related:
$$\text{Confidence Level} = 1 - \alpha$$

For a 95% confidence interval: $\alpha = 1 - 0.95 = 0.05$

## Applications in Machine Learning

### 1. Model Validation

Hypothesis testing confirms that a model's performance on training data generalizes to new, unseen data. When comparing two models, hypothesis testing determines if the difference in performance metrics (accuracy, precision, F1-score) is statistically significant.

**Statistical Framework for Model Selection:**

- **Null Hypothesis ($H_0$):** There is no significant difference in performance between Model A and Model B
- **Alternative Hypothesis ($H_1$):** There is a statistically significant difference in performance
- **Significance Level ($\alpha$):** Typically 0.05

**Decision:** If $p\text{-value} \leq \alpha$, reject $H_0$ and conclude one model is likely superior.

### 2. Feature Selection

Hypothesis testing helps identify which features have significant relationships with the target variable:

- **Chi-square test** for categorical variables
- **Correlation coefficients** for continuous variables

This filters out irrelevant or redundant features, streamlining model-building.

### 3. A/B Testing

A/B testing compares two versions of a model to determine which performs better in real-world scenarios. Hypothesis tests analyze A/B test results to validate whether observed differences in KPIs are statistically significant.

### 4. Hyperparameter Tuning

Hypothesis testing evaluates whether changes in hyperparameters lead to significant improvements in model outcomes.

### 5. Model Comparison Using Statistical Tests

### Comparing Two Models: Z-Score Test for Proportions

The z-score for comparing two proportions (accuracies):
$$z = \frac{p_1 - p_2}{\sqrt{\hat{p}(1-\hat{p})(\frac{1}{n_1} + \frac{1}{n_2})}}$$

Where:
- $p_1, p_2$ = accuracies of models 1 and 2
- $\hat{p} = \frac{n_1 p_1 + n_2 p_2}{n_1 + n_2}$ (pooled proportion)

**Limitation:** This test assumes independence between predictions, which is often violated in practice.

### Information Criteria for Model Selection

**Akaike Information Criterion (AIC):**
$$AIC = 2k - 2\ln(\hat{L})$$

**Bayesian Information Criterion (BIC):**
$$BIC = k\ln(n) - 2\ln(\hat{L})$$

Where:
- $k$ = number of parameters
- $n$ = sample size
- $\hat{L}$ = maximum likelihood

**Decision Rule:** Lower AIC/BIC indicates better model (balances fit and complexity)

## Model Selection Strategies

### 1. Statistical Inference for Model Selection

Treats model selection as a formal hypothesis-testing problem:

- **Information Criteria (AIC/BIC)**: Balance goodness-of-fit against complexity
- **Likelihood Ratio Tests**: Compare nested models to determine if additional variables significantly improve performance

### 2. Model Inference for Selection (Performance-Based)

Modern ML approach using prediction performance on unseen data:

- **Hold-out Validation**: Test on a separate validation set
- **Cross-Validation**: Repeat inference across different data folds for robust selection
- **Bayesian Optimization**: Use probability models to predict which hyperparameters perform best

### 3. Practical Model Comparison Workflow

```
1. Define candidate models
2. Use cross-validation to obtain performance distributions
3. Apply appropriate statistical test (e.g., paired t-test)
4. Check p-value against significance level
5. Select model based on statistical evidence
```

## Philosophical Approaches

### Fisher's Approach
Fisher's **p-value** is the probability that a result as extreme as what was found occurs, assuming $H_0$ is true. Fisher argued that $p$ measures the significance of the data.

### Neyman-Pearson Approach
Provides a decision rule for behavior over the long term. Rather than establishing truth in $H_1$ or $H_2$, it focuses on providing a decision rule that follows the truth as much as possible on average across many decisions.

## Key Takeaways

✅ **Hypothesis testing** provides a formal framework for making data-driven decisions

✅ **Statistical inference** enables conclusions about populations based on sample data

✅ **Type I and Type II errors** must be balanced based on problem context

✅ **Confidence intervals** quantify uncertainty in parameter estimates

✅ **Machine learning** extensively uses hypothesis testing for model selection, feature selection, A/B testing, and validation

✅ **p-values** must be interpreted correctly: they measure evidence against $H_0$, not the probability that $H_0$ is true

✅ **Cross-validation** provides robust performance estimates for comparing models

✅ **McNemar's test** is particularly valuable for comparing models on a single test set without expensive retraining
